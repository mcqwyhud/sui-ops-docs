# 动态节点自动扩缩容配置教程

本指南向运营者介绍集群动态节点调度，针对不同 VPS 类型与地区实现全自动弹性扩缩容与计费优化。  
**当前版本采用房间制实时分配，动态节点不预留离线用户，资源利用率可达理论 100%。**

---

## 1. 扩缩容核心单元与命名规范

系统以 `VPS类型 + 中文地区` 作为独立扩缩容单元（唯一标识，如 `dynamic@美国 洛杉矶`）。各单元之间的负载统计、实例池与扩缩容决策完全独立，互不干扰。

### 地区匹配与映射规则
* **动态扩容 VPS**：系统调用云商 API 创建实例时，根据机房区域（如 `lax`、`nrt`、`sgp`）自动生成 `国家 城市` 中文标签（如 `美国 洛杉矶`）。
* **手动创建的固定 VPS**：必须在总控后台将地区手动配置为相同的 `国家 城市` 格式。固定 VPS 作为该地区的底座容量，与动态 VPS 一同计入总负载统计。
* **机房代码映射**（触发扩容时）：
  - `美国 洛杉矶` / `洛杉矶` ➔ `lax`
  - `日本 东京` / `东京` ➔ `nrt`
  - `新加坡` ➔ `sgp`

---

## 2. 容量与负载统计（房间制核心）

- **动态 VPS 仅统计实时在线人数**，用户断开后立即释放容量，不保留任何“已分配”记录。
- **每个动态 VPS 的容量上限（`maxUsers`）即为最大在线人数**，在该值范围内可灵活接纳新连接。
- 负载率计算公式：  
  `负载率 = 当前在线总人数 / 动态 VPS 总容量上限（含固定 VPS 提供的底座容量）`
- 因此，即使总用户量远超在线人数，只要在线人数未触顶，负载率始终保持在健康水平，无需频繁扩容。

---

## 3. 扩容与缩容执行逻辑

每个检测周期（默认 30 秒），总控通过 cache2k 内存聚合所有节点的实时在线人数，计算负载率。

### 扩容机制（渐进式抗峰）
* **触发条件**：某单元负载率达到或超过设定的扩容阈值（如 90%）。
* **执行行为**：每个周期内，该单元最多调用云商 API 创建一台新 VPS。若上线后负载仍超标，下个周期继续创建，直至负载回落。
* **冷却保护**：扩容成功后进入冷却期（默认 120 秒），期间该单元不再重复扩容。

### 缩容机制（计费优化排空）
* **触发条件**：全局负载率持续低于设定阈值（如低于 30% 持续 5 分钟）。
* **目标选择**：优先选择**计费周期即将结束（如 Vultr 整点前 10 分钟）**且**在线人数最少**的动态 VPS 执行回收。固定 VPS 绝不参与缩容。
* **无感迁移流程**：
  1. 目标 VPS 进入 `排空` 状态，总控停止向其分配新用户。
  2. 存量在线用户由客户端在后台平滑迁移至同地区其他正常 VPS（基于房间制，用户直接切换房间）。
  3. 迁移完成或排空超时（默认 600 秒）后，调用 API 销毁实例，确保不产生额外计费。

## 4. 扩缩容控制台参数详解

登录 SUI-Ops 后台，进入 `系统设置` -> `动态扩缩` 进行配置：

| 参数名称 | 默认值 | 说明                        |
| :--- | :--- |:--------------------------|
| **检测间隔** | 30 秒 | 负载采集与决策周期                 |
| **单 VPS 最大在线人数** | 10 人 | 动态 VPS 允许承载的最大在线人数（即容量上限） |
| **最小/最大动态 VPS 数** | 0 / 10 | 限制单地区动态实例池边界，防 API 滥用     |
| **扩容触发阈值** | 90% | 负载率超过此值时触发创建新 VPS         |
| **缩容触发阈值 / 持续时间** | 30% / 300秒 | 负载低于此值并持续 5 分钟后触发排空       |
| **计费周期 / 提前销毁缓冲** | 60分钟 / 5分钟 | 按小时计费时，在整点前 5 分钟完成实例销毁    |

---

## 5. 调度模式说明

运营者可针对不同节点单元单独划分时间段策略：

* **全天候模式**：24 小时全自动扩缩容。
* **指定时段模式（如晚高峰 18:00-23:00）**：仅在活跃时段内允许扩容；非活跃时段系统停止扩容，并逐步排空、回收所有动态 VPS，降至最小保留数。
* **时区说明**：调度策略基于 VPS 物理机所在本地时区执行。如需调整，请联系技术支持。
