# 中转与直连部署指南

根据您的业务架构，Sui-Ops 支持两种部署模式。请根据实际需求选择，以确保系统性能与安全的最优化。

---

## 1. 直连模式 (Standalone)
**适用场景：** 单机直接作为出口，或不经过中转的流量转发。

*   **部署要求：**
    *   机器上需安装 Sui-Ops 子控。
    *   该机器具备完整的管理功能，向总控汇报实时状态。
*   **管理权限：** 完全受 Sui-Ops 调度。

## 2. 中转中继模式 (Relay-Linked)
**适用场景：** 隐藏落地机真实 IP，或通过中转机优化全球网络连接。

### 核心逻辑：
该模式将系统分为 **[中转机]** 与 **[落地机]**，实现管理与流量分离。

*   **中转机 (Gateway)：**
    *   **必须安装 Sui-Ops 子控**，作为流量的总入口与调度核心。
    *   负责管理所有下挂的落地机配置及流量转发规则。
*   **落地机 (Worker)：**
    *   **严禁安装 Sui-Ops 任何组件**。
    *   落地机仅作为“纯粹的流量出口”。
    *   操作方法：直接在中转机的总控后台添加落地节点信息即可，无需任何代理程序或面板介入，最大程度降低落地机被探测风险。
    *   如果您有特殊需求，需要在总控显示到vps但不受vps管理，请一定将其vps类型设置为一个特别的自定义类型，并不要与任何套餐 cdk关联

---

## ⚠️ 部署注意事项 (重要)

> **如果您采用中转架构：**
> 1. **落地机纯净化：** 请勿在落地机运行 子控 安装命令。保持落地机环境高度简洁，只运行必要的转发服务。
> 2. **管理解耦：** 后台仅监控“中转机”状态。落地机的健康状态需通过中转机的转发链路进行探测。

---

### 💡 快速决策表

| 功能特性 | 直连模式 (Standalone) | 中转模式 (Gateway + Worker) |
| :--- |:------------------| :--- |
| **Sui-Ops 子控** | 必须安装              | **仅中转机安装** |
| **落地机环境** | N/A               | 纯净、无任何管理面板 |
| **安全性** | 标准                | **极高 (落地机隐身)** |
| **维护难度** | 节点级维护             | 集中式维护 (在中转机操作) |
