# 多级分销与管理员配置教程

本指南向运营者介绍如何利用 SUI-Ops 的无限层级架构搭建代理分销体系，配置数据权限隔离以及管理员日常安全管理。

---

## 1. 代理层级架构与权限隔离

由于 SUI-Ops 总控后端出于安全和闭环控制考量，**完全不支持开放式自主注册**，所有的管理员账户均必须由上级或主管理员在后台手动添加创建。系统支持构建的多级分销网络底层逻辑如下：

### 无限层级嵌套体系
* **全手动开户**：所有下级分销渠道、代理账户均由其直属上级管理员手动录入用户名、配置初始密码。创建下级管理员时，系统会自动在底层绑定并锁死其对应的唯一上级关系链。
* **层级关系**：支持 管理员 → 下级管理员 → 下下级管理员 的无限层级树状嵌套。
* **严格权限分配校验**：管理员在手动开户或为下级分配权限时，系统会自动进行强校验，管理员只能分配自身已经拥有的权限给下级管理员，无法跨权限或越权授予自身不具备的功能。

### 绝对数据权限隔离与用户管辖
* **管辖范围限制**：每个管理员登录系统后，只能查看、管理和修改自己以及自身下级链条管辖范围内的下级管理员数据。
* **用户看视隔离**：在用户管理层面，每个管理员只能看到属于自己直属的用户以及其下级管理员所管辖的用户，下级管理员以此类推向上递归隔离，无法查看或干涉非自身链条下的任何终端用户。
* **越权防御**：非同链或跨条线的管理员与用户数据在数据库和接口层完全隔离，确保大户代理之间的商业机密安全。

## 2. 管理员账户创建与日常运维

请登录 SUI-Ops 总控后台，进入 管理员管理 页面进行维护。

### 基础运维操作
* **开户创建**：点击新建管理员，手动录入基本信息。分配权限时，勾选框仅会展示当前操作人自身拥拥有权限的选项，保存后系统自动绑定当前操作人为其上级。
* **账户检索**：支持通过用户名、昵称、状态、管理员级别等条件进行多级筛选。
* **安全锁死与清理**：当启用/禁用某个管理员账户时，若选择禁用，总控将自动强制清理该管理员名下的所有活跃登录 Token，阻断其继续访问。
* **风控解锁**：系统内置登录失败锁定机制。若代理因密码错误导致账户被锁，上级管理员可通过密码重置或账户解锁功能为其恢复状态。
* **个人中心**：每个管理员均拥有独立个人中心，可自主修改昵称、修改密码以及设置专属客服链接，当设置专属客服后 客户端用户点击"联系我们"时会自动跳转到归属管理员客服链接。

## 4. 自动化获客：推广码体系

为进一步降低运营门槛，系统内置了基于 URL 参数的自动化获客绑定机制，这是您分销网络的“第一触点”。

### 推广码获客逻辑
- **自动绑定机制**：系统通过 URL 参数 `referral=数字ID` 识别推广关系。当潜在用户通过带有此后缀的门户链接访问时，前端系统将自动留存推广 ID。
- **关联注册**：当用户在门户完成注册时，系统会自动提取记录的 ID，将其直接绑定至对应的管理员名下。
- **投放建议**：
    - 您可以进入 [管理员管理] 页面查看您的专属 ID。
    - 将生成的推广链接（如 `http://域名/portal/?referral=您的ID`）投放至社交媒体或卡盟渠道。
- **注意事项**：
    - 这里的域名是门户域名，不是后端管理系统的域名，不要混淆了。
    - 绑定遵循“首登即绑定”原则，注册完成后关系即刻锁定，不可后期补录。
    - 若用户未携带推广码直接注册，系统默认为无上级，无法享受自动分润。

### 余额变动审计
* **资金调整**：上级管理员拥有对直属下级账户进行增加余额、扣减余额、或者直接设置余额的权限。
* **强审计日志**：每一次余额变更均属于敏感操作，系统强制要求必须填写变更原因。变更记录、操作人、原因及时间将永久写入系统审计日志。

### 佣金比例与收款结算
* **自定义提成比例（commissionRate）**：可为每一个管理员单独设置个性化的提成比例。该比例直接参与 CDK 激活或网关支付回调时的级差提成算法计算。
* **USDT（TRON）收款地址配置**：管理员可在后台配置自己的波场网络（TRC-20）加密货币收款地址，用于日常的佣金结算与提现对接。
