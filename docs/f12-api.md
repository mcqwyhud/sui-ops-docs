# 更多高阶接口自行查看说明

> **核心原则：轻量高效，聚焦变现。**

SUI-Ops 的核心设计初衷，是利用高可用的自动化集群和完美的商业闭环，**帮运营人员（站长）省下繁琐的运维精力，把更多的时间留给市场营销与商业变现**，而不是深陷在技术泥潭里做功能拓展。

因此，官方文档目前仅对涉及资金流、自动化扩缩容的核心接口（认证、发卡、VPS、节点监控）进行规范编排。我们不建议在现阶段引入过多低频、冗余的 API，这会盲目增加您外部系统的业务复杂度，分散您的核心运营精力。

---

## 应急对接指引

本系统采用完全的前后端分离架构，网页后台的所有操作均通过 API 与后端交互。

如果现有的核心 API 依然满足不了您的特殊业务，且您急切需要进行深度定制：

* **自行拓展**：请直接登录总控后台，自行查看对应的网络请求（Network）并抓包对接。
* **鉴权规则**：自行调用的接口，在 Header 中同样必须携带 `Authorization: Bearer <您的API管理员Token>`。

随着产品后续的迭代，官方也会本着“极简、实用”的原则，逐步补充更多能切实提升运营效率的规范 API。
