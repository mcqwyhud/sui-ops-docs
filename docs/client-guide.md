# 客户端使用与多端下发指南（运营实操）

本指南面向总控运营者，聚焦**具体操作步骤、分发规范、客诉处理话术及高阶企业配置**。若需了解客户端功能设计理念，请先参阅《[客户端生态](client-ecosystem.md)》。

---

## 📥 1. 双端安装包分发规范（Windows / Android）

总控部署完成后，我们会为您交付专属定制的客户端安装包（.zip 与 .apk）。

- **建议分发渠道**：将安装包上传至您的门户网站下载页，或托管至大带宽对象存储（如阿里云 OSS、Cloudflare R2），确保用户高速下载。
- **安全提示配置**：由于全内存运行机制，部分魔改杀毒软件可能产生误报。建议在门户下载页显著位置提示：*“如遇 Windows 安全中心或第三方杀软误报，请添加信任区后运行。”*

## 🍏 2. 全生态通用订阅链接下发（ios / 软路由 / 开源客户端）

系统前端门户提供的“一键获取订阅”功能，本质上下发的是**标准 s-ui 订阅链接**。运营需熟知以下下发与配置指引，以便解答用户咨询：

### 2.1 ⚠️ 关键运营提示：动态节点的不可获取性
*   **兼容性边界**：动态调度功能完全依赖于“自研客户端”与“后端实时链路监测”的深度联动。因此，**动态节点池不支持生成通用订阅链接**。
*   **用户侧感知**：若用户购买的套餐包含动态节点权限，在门户网站获取 ios/软路由订阅时，系统将**自动过滤掉所有动态节点**，仅下发该账户权限下的“固定节点”。
*   **运营策略**：
  *   **引导方案**：请务必引导需要使用“智能调度”功能的用户下载并安装 **SUI-Ops 自研 Unity 客户端**。
  *   **ios 变现策略**：如果您有大量 ios 高净值用户，建议您在总控后台为其配置 **“固定出口”套餐**，确保他们能够通过通用订阅链接获取到稳定的节点列表，从而规避动态调度带来的配置缺失问题。
  *   或使用中转机自行做负载均衡

### 2.2 ios 用户配置指引（话术模板）
- **Step 1**：引导用户在海外 App Store 登录国外 Apple ID，下载通用客户端（如 **sing-box** 或 Shadowrocket）。
- **Step 2**：指导用户登录您的门户网站 > 进入“个人中心” > 点击“获取 ios 订阅” > 复制生成的订阅链接。
- **Step 3**：打开 sing-box 等客户端，点击“添加订阅” > 粘贴链接 > 点击更新即可完成节点同步。

### 2.3 软路由 / NAS 与极客用户配置
- **操作指引**：告知用户直接复制该标准订阅链接，无缝导入 **OpenWrt（OpenClash/PassWall/SSR-Plus）、iStoreOS** 或各类 NAS 容器（如 Docker 版 Clash）中。
- **运营注意**：软路由和第三方客户端无法被自研客户端的“单账号在线锁定”逻辑锁死（详见第 4 节企业绕过方案）。

### 2.4 套餐隔离建议（重要）
> 由于 ios/软路由订阅均为原生 s-ui 协议，若您希望针对此类用户开启或关闭“多端共享检测”，建议在配置 VPS 节点时，**将 ios/软路由专用套餐与普通自研客户端套餐的节点 VPS 区分开来**。具体检测开关配置请参考《[风险控制与防共享](risk-control.md)》及《[VPS 管理](vps-config.md)》。

## 🚫 3. 零售基础运营：单在线排他性客诉处理

自研客户端默认开启“单账号唯一登录”限制。遇到以下常见客诉，请按标准话术处理：

- **场景一：更换设备登录失败**
    - *用户反馈*：“手机连着，电脑登不上。”
    - *标准答复*：“尊敬的客户，为确保账号安全与带宽质量，本平台实行单设备在线制。请您先手动断开前一个设备的连接（或直接关闭客户端），等待 3-5 秒后端清理缓存后，新设备即可正常登录。”

- **场景二：被误判合租频繁断线**
    - *用户反馈*：“我和室友轮流用，怎么老掉线？”
    - *标准答复*：“检测到您的账号存在多端并发连接，系统已自动拦截。本平台严禁账号共享/合租，如需多端同时在线，请升级至‘企业尊享版’或购买独立多端套餐。”（趁机转化高价值用户）

## 🏢 4. 企业级进阶玩法：软路由不限并发绕过配置

针对企业工作室、软路由大户等高净值客户，您可以通过底层预留机制解除多端限制，并将其打包为高价套餐售卖。

**具体操作步骤如下：**
1. **关闭检测**：登录子控后台配置，默认安装配置路径：/opt/sui-agent/config/nodeRisk.json，手动**关闭“多端使用检测（共享检测）”**功能开关。
2. **下发订阅**：让该企业用户统一使用第 2 节中提到的**原生 s-ui 订阅链接**，直接配置到他们的软路由（如 OpenWrt）或原生开源客户端（如 Clash）中。
3. **商业包装**：由于关闭检测后，该订阅不再受单账号在线锁限制，可实现不限设备数并发。建议您将此能力打包为 **“企业尊享版 / 软路由大户套餐 / 不限并发包”** 进行单独溢价售卖。

## ⚡ 5. 后台节点调度标签配置指引

在总控后台配置节点时，运营需根据售卖策略打上对应标签，以匹配客户端的前端显示逻辑：

- **智能调度标签**：激活后，该节点将自动归入区域节点池（如“韩国”）。用户客户端切换至“智能模式”时，由总控实时分配该池内的节点，用户下线后资源自动释放。
- **固定出口标签**：适用于对 IP 稳定性要求极高的业务。打上此标签后，用户在客户端切换至“固定模式”，系统将为其绑定该物理节点的固定名额，出口 IP 永不变化。

## 💡 附录：常见运营 FAQ

**Q：用户反馈 ios 订阅链接导入后节点不显示？**  
A：请检查用户是否在门户网站完成了套餐购买且账户未过期；若已过期，订阅链接将返回空节点列表。

**Q：企业用户关闭多端检测后，还能否统计流量？**  
A：可以。无论是否关闭检测，流量统计与到期时间均正常上报总控后台，不影响计费逻辑。
