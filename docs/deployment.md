# 系统部署指南

通过自动化部署脚本，您可以在 3 分钟内完成 SUI-Ops 子节点vps环境的初始化并接入总控。

## 1. 节点vps环境要求
为保证节点vps逻辑服的稳定运行，我们对 VPS 配置要求如下：

* 操作系统: Ubuntu 22.04 LTS
* 最低配置: 1 vCPU / 512MB RAM
* 网络需求: 需具备固定 IPv4，且开放 TCP/UDP 相关业务端口

---

## 2. 一键自动化接入流程

### 第一步：执行安装脚本
请使用 root 用户登录您的节点 VPS，直接复制并执行以下命令：

```bash
bash <(curl -Ls https://raw.githubusercontent.com/mcqwyhud/sui-group/main/suiAndAgent_linux_install_v2.0.sh)
```

### 第二步：配置连接参数
脚本运行后，系统会提示您输入以下必要信息，请提前准备好：

1. 部署 Token: 输入您的私密仓库访问令牌（用于授权拉取逻辑服镜像）。
2. 网关密钥: brokerKey: 总控后台可查看。
3. 总控地址: brokerHost: 总控的ip或域名。
4. 网关端口：brokerPort: 没改的话默认即可[10200]
5. 其它的根据需求更改,但需要确保正确

### 第三步：后台配置vps

当脚本提示“安装成功”后，您的vps已自动注册至总控后台。请根据以下步骤激活vps功能：

1. **登录后台**：访问您的 SUI-Ops 总控后台 (https://<您的ip>:8080/vue)。
2. **定位vps**：进入 **[VPS管理]** 页面，找到刚才部署好的vps,可通过pid，ip等区分。
3. **完成配置**：点击“编辑”，设置该vps的 **[VPS类型]** 及相关参数。

> **⚠️ 关键提示**：
> 该vps的 **[VPS类型]** 必须与您的 **[套餐配置]** 中的标签完全匹配，否则 CDK 将无法正常发放权限。
> 详细的操作步骤与参数逻辑，请参考：**[VPS 管理使用指南](vps-config.md)**。

---

## 3. 节点进阶管理与配置规范

### 节点配置快速访问
接入完成后，您可以直接通过浏览器访问节点的 s-ui 面板进行底层调试：
- 默认访问地址: http://<您的节点IP>:2095/app
- 提效建议: 您可以预先配置好一套“s-ui 通用模板”，将配置完成后的 .db 文件备份保存。后续部署新节点时，直接上传该文件并微调参数，即可跳过重复的配置流程。
- s-ui的入站命名格式：国家1-xxx , 国家 城市1-xxx 。这里的数字和xxx可以任意填 方便自己区分

> **安全提示**: 备份的 .db 文件包含核心密钥，请务必妥善保管，切勿上传至任何公共仓库。

### 运营管理边界说明
为了保证总控系统数据的权威性与一致性，请务必遵守以下操作规范：

- **用户数据管理**: 请勿在 s-ui 面板中直接添加或修改用户。总控系统仅以自身数据库为准，会忽略 s-ui 面板中手动添加的无效用户数据。如需进行用户增删改查，请统一在 SUI-Ops 总控后台操作。
- **节点特定配置**: 若需调整节点底层的协议细节或特定链路参数（如修改节点端口等），请直接进入对应 VPS 的 s-ui 面板进行针对性修改。

## 4. 节点侧风控与防共享配置 (nodeRisk.json)

本系统在子节点vps集成了生产级的“账号共享”与“多端高频并发连接”检测机制。采用“底层行为评分”、总控“最终决策”的分布式架构，支持根据不同代理协议动态扩展权重。

若您需要开启或调整该 VPS 的防白嫖风控阈值，请直接修改该vps环境下的 nodeRisk.json 配置文件：

{
    "enable" : false,
    "windowMs" : 10000,
    "ipWeight" : 10.0,
    "ipWarn" : 3,
    "ipLimit" : 6,
    "connWeight" : 1.0,
    "connWarn" : 10,
    "connLimit" : 25,
    "burstWeight" : 25.0,
    "burstConnThreshold" : 8,
    "burstIpThreshold" : 3,
    "finalScale" : 1.0,
    "protocolWeights" : {
        "wg" : 1.2,
        "hysteria2" : 1.5,
        "trojan" : 1.0,
        "wireguard" : 1.2,
        "shadowsocks" : 1.0,
        "tuic" : 1.7,
        "vless" : 1.0,
        "vmess" : 1.0
    },
    "defaultProtocolWeight" : 1.0
}

核心参数详解：
- enable: 是否开启当前vps的防多端共享检测（设置为 true 开启，false 关闭。注：若开通极客/企业多端不限并发套餐，可将此项设为 false 配合原生开源客户端绕过）。
- windowMs: 行为统计的时间窗口（单位：毫秒），默认 10000 毫秒（10秒）。
- ipLimit / connLimit: 触发风控判定的单账号最大 IP 变动上限与最大并发连接上限。
- protocolWeights: 协议插件系统。支持针对不同协议单独调整风险权重系数（例如对于 Hysteria2 或 Tuic 等 QUIC/UDP 类高并发协议，预设了更高的风险权重，无需修改源码，在此处增删即可）。
