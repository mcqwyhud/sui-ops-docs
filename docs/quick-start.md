# 运营起步核对清单

恭喜您成为 SUI-Ops 生态运营者！为了帮助您以最快速度完成从部署到盈利的转换，请依照以下路径进行核对：

---

## 1. 基础设施部署阶段
- [ ] **模式选择**：在部署前，务必参考 [中转与直连部署指南](node-mode.md) 确定您的架构模式。
- [ ] **节点部署**：
    - **直连节点**：按照 [系统部署指南](deployment.md) 完成子控安装。
    - **中转/落地节点**：仅需在中转机部署子控；落地机保持纯净，无需任何操作。
- [ ] **节点认领**：进入后台 [VPS 管理](vps-config.md)，查看节点是否已自动注册（仅直连及中转机），并点击“编辑”配置地理位置及资源配额。
## 2. 商业化定价阶段
- [ ] **定义套餐**：在 [套餐配置](plan-config.md) 中新建您的售卖产品，明确流量、时长及对应的 [VPS 类型] 标签。
- [ ] **支付关联**：配置订阅链接，决定使用系统内嵌支付还是跳转至第三方平台。

## 3. 销售与分销部署
- [ ] **发放 CDK**：在 [CDK 管理](cdk-config.md) 中创建第一批测试码，测试购买流程是否顺畅。
- [ ] **配置分销链**：在 [多级分销机制](distribute.md) 中，为您的合伙人/分销商创建账号，并设定好级差提成比例。

## 4. 运营安全与监控
- [ ] **开启防护**：检查 [风险控制与防共享](risk-control.md) 设置，根据需要开启多设备同时在线限制。
- [ ] **监控设置**：参考 [节点健康与自动迁移](node-health.md)，确保节点故障能触发自动割接。

## 5. 客户交付
- [ ] **客户端**：查看 [客户端使用与多端下发](client-guide.md)，使用客户端登录用户账户检查节点是否正确。

---

> **💡 运营专家小贴士**：
> 如果您在运营过程中遇到任何 CDK 无法激活、节点权限异常的情况，请首先核对 [VPS 管理](vps-config.md) 中的“类型标签”与 [套餐配置](plan-config.md) 是否完全匹配。这是 90% 的运营者遇到的首要配置问题。
